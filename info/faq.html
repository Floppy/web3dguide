What do I need to write VRML?

You only need a text editor. This is all explained in the Getting Started VRML97 tutorial. However, if you want to start serious content creation, there are a few tools that I can recommend:

  1. A text editor, for raw content creation. I do 95% of my VRML creation in VrmlPad. There are links to others on the tools page.
  2. A paint program, for textures. I recommend Paint Shop Pro from JASC.
  3. A VRML validator, such as Chisel from Trapezium This is vital for your content creation, especially if you're using a text editor to do it. Browsers don't give helpful error messages, but this will. If you have a problem in your world, this will find it. It will also optimise your files and make them more efficient. It also does GZIPing of VRML files for you.
  4. A 3D Modeler, for the more complex IndexedFaceSet stuff. For this, I use Nendo, from Winged Edge Technologies.
  5. A Java compiler, for scripting. I would recommend Sun's own JDK (Java Development Kit), available from java.sun.com. It's free, has everything you need and has plenty of handy features.
So, what more do you need to create great VRML? Well, imagination, of course, but there's no web site for that!

Can I use BMP/GIF/etc files as textures?

In general, probably not. Your browser might support it, but it's not standard. Stick to the standard .JPG and .PNG files.

I've written my world, but when I load it into the browser I get a black screen. What's wrong?

This happens because the browser doesn't recognise what it has been told to load. This is usually because you've forgotten to put the header at the top of the file. Remember, every VRML file must have the magic header #VRML V2.0 utf8 at the top of the file. If you have got this line, it is possible that the file is not in plain text format. One of the more frequent reasons I find for this is that you've written your code in Wordpad and saved it as a Microsoft Word document. If you just get a blank window when you try to view your worlds, load them into Notepad and make sure they're plain text only. Better still, always use a plain text editor to write VRML!

How can I write information to the VRML console?

Some browsers have a function built into their Java and ECMAScript interpreters that allow a script to write values into the VRML Console (where errors and other messages are displayed). Some also allow you to write to the browser's Java console. The functions that I know of so far for the different browsers are listed below. There is also more information available in the debugging workshop.

CosmoPlayerprint(string)
